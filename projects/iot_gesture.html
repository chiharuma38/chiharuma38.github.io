IoT Gesture Recognition System

Project Summary

This IoT-based fertilizer spiller system uses ESP32 microcontrollers and machine learning to recognize user hand gestures, enabling both remote manual and automatic gesture-based control of a model vehicle for smart farming applications. By integrating an MPU6050 sensor, custom neural network, and ESP-NOW wireless protocol, the project demonstrates how embedded ML and IoT can automate agricultural processes for greater efficiency and ease of use.

  • Diverse Control: Manual mode uses button presses; automatic mode uses gestures (Spin, Back, Front, Left, Right, Stop) detected by the MPU6050 sensor.
  • Machine Learning: Neural network trained on over 1,200 labeled gesture samples, running real-time on ESP32 hardware.
  • Robust Wireless: ESP-NOW protocol ensures low-latency communication between controller(s) and the car, outperforming Bluetooth for this application.
  • Field-Tested: System response time less than 1 second; reliable control in both classroom and field environments up to 220 meters.
  • Modular Design: System supports easy switching between manual and automatic (gesture) modes for flexibility in operation.

Technical Highlights

  • Data Collection: Six gestures mapped to output numbers (Spin-0, Back-1, Front-2, Left-3, Right-4, Stop-5); each motion captured with labeled IMU data.
  • Model Training & Deployment: Data processed with Python, model trained using TensorFlow/Keras, converted and deployed to ESP32 via TensorFlow Lite for Microcontrollers.
  • Pin Configuration: Detailed pin mapping for manual/automatic modes, motors, and sensors for reproducibility and troubleshooting.
  • Software: Modular C++ code for ESP32, Python for ML, and clear documentation for gesture-to-action mapping.

Lessons Learned

  • High-quality, well-labeled data is critical for robust gesture ML models; increasing the dataset size greatly improved accuracy.
  • Deploying ML to microcontrollers requires careful model optimization and understanding hardware constraints (RAM, flash, CPU).
  • ESP-NOW provided more stable, faster communication than Bluetooth for multi-microcontroller setups.
  • Divide-and-conquer team management and open communication streamlined troubleshooting and accelerated progress.
  • Consulting official documentation and community forums was key to overcoming integration and deployment hurdles.
