Advanced Safe System

Secure safe with alphanumeric keypad authentication, break-beam intrusion detection, LED and siren alerts, and servo-controlled locking — built on STM32 microcontroller.

STM32 Embedded Systems Security C Programming ARM Cortex-M

Project Overview

Designed and built a multi-layer security safe system using an STM32 ARM Cortex-M microcontroller. The system integrates multiple input and output peripherals to provide layered physical security with real-time alerts.

  • Authentication: Alphanumeric keypad entry with configurable PIN — incorrect attempts trigger lockout and alarm.
  • Intrusion Detection: Break-beam IR sensor monitors the safe opening — unauthorized access triggers a siren and LED alert.
  • Servo Lock: Servo motor controls the physical bolt mechanism, driven by PWM signal from the STM32.
  • User Feedback: LCD display shows status messages and countdown timers for lockout periods.

Technical Highlights

  • STM32 HAL library used for GPIO, UART, PWM, and timer peripherals.
  • State machine architecture managing authentication, locked, unlocked, and alarm states.
  • Interrupt-driven keypad scanning for responsive input without polling delays.
  • Debouncing implemented in firmware for reliable keypad and sensor readings.

Lessons Learned

  • Peripheral integration requires careful attention to timing — servo PWM frequency must not interfere with other timer-based peripherals.
  • Real-world security systems need fail-safe defaults — the safe locks automatically on power loss.
  • Embedded state machines make complex behavior manageable and testable.
