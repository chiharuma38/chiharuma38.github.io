FPGA Catch-the-Falling-Object Game

Real-time game implemented entirely in VHDL on an FPGA board, with random object generation, collision detection, and score display via VGA output.

FPGA VHDL Digital Design VGA Real-time Systems

Project Overview

Designed and implemented a complete real-time game in VHDL running on an FPGA development board. The game renders falling objects via VGA output, detects collisions with a player-controlled catcher, and tracks and displays the score — all in hardware logic with no software processor involved.

  • Random Object Generation: Linear feedback shift register (LFSR) used to generate pseudo-random spawn positions for falling objects.
  • Collision Detection: Pixel-level bounding box collision logic implemented in VHDL combinational logic.
  • VGA Output: Custom VGA controller generating 640x480 display with sync signals and color output.
  • Score Display: Seven-segment display driver implemented to show live score updates.

Technical Highlights

  • Entire game logic implemented in synthesizable VHDL — no embedded processor or software.
  • State machine architecture managing game states: idle, playing, game over.
  • Clock dividers used to generate precise timing for VGA sync and game update rates.
  • Simulated and verified using ModelSim before hardware deployment.

Lessons Learned

  • Hardware description languages require a fundamentally different mindset from software — everything runs in parallel.
  • Timing constraints and clock domain crossing are critical to get right before synthesis.
  • Simulation is essential — debugging on hardware without simulation is extremely difficult.
