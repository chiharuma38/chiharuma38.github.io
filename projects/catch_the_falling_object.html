Project Summary

This project is a real-time, interactive game implemented purely in VHDL on an FPGA board, with VGA display output. The player controls a basket to catch falling objects (balls), with randomly generated positions and increasing difficulty. The design features modular hardware programming, collision detection, score tracking, and robust state machine control for a seamless gaming experience and instructional demonstration of digital logic systems.

  • Full Hardware Implementation: Designed and coded entirely in VHDL, running on a DE10-Lite FPGA development board.
  • Real-Time VGA Output: Custom VGA controller produces smooth graphics on a 640x480 display; all gameplay visuals are generated by hardware.
  • User-Controlled Basket: Players use FPGA board switches to move the basket horizontally and catch falling objects.
  • Random Ball Generation: A Linear Feedback Shift Register (LFSR) generates unpredictable ball positions for dynamic gameplay.
  • Collision Detection & Scoring: Hardware logic checks if the basket catches a ball and updates the score, shown on a 7-segment display.
  • Finite State Machines (FSM): Control both the game logic and the score system, ensuring reliability even during fast-paced play.

Technical Highlights

  • Modular Design: Separated VHDL modules for VGA sync, basket movement, ball generation, collision, and display for easier testing and debugging.
  • Resource Efficient: Ball rendering uses combinational logic for each pixel, avoiding memory-intensive approaches.
  • Debounced Scoring: FSM prevents double-counting of catches; all input signals are synchronized to the horizontal sync pulse for stability.
  • High Catch Accuracy: Achieved ~99% collision detection accuracy in testing, with rare errors only at edge pixels.
  • Visual Feedback: LEDs on the FPGA board indicate "catch" and "miss" events for clear, real-time feedback.

Lessons Learned

  • Modular, testable design is essential for complex hardware projects—each VHDL module was developed and debugged independently before integration.
  • Careful timing analysis and synchronization (all modules tied to VGA horizontal sync) prevent display glitches and ensure consistent game logic.
  • FSMs are powerful for robust control and debouncing, especially with fast or noisy inputs.
  • Efficient combinational logic can replace memory-heavy solutions for real-time graphics and game state.
  • Iterative improvement: initial designs for basket movement and random number generation were refined for better user experience and randomness.
